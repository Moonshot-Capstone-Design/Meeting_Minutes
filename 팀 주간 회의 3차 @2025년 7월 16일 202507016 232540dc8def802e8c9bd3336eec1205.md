메모

받아쓰기

트레일러 부착으로는 사각지대가 존재한다. 그래서 그걸 내가 보완할 방법을 생각하면 좋겠다고 가져왔다. 그러면 일단은 트레일러에 부착을 해서 어떤 센서를 쓸 것이고 어떤 식으로 운영을 시킬 것인데, 사각지대가 존재한다는 것이다. 그러니까 트레일러를 부착한다고 하더라도 아래쪽 벽 아랫면에 붙일 것인지 옆면에 붙일 것인지에 따라서 또 다르다. 그 범위를 한 번에 다 커버하기는 어렵다.

범위가 너무 넓다 보니 제한이 있다. 만약 트레일러는 검사 장비를 이동시키는 도구일 뿐이고, 중요한 것은 어떤 센서를 써서 어떤 방식으로 검사를 하고 보완할지라는 점이다. 어떤 센서를 쓸 것인지 들어가 보면 논문 같은 것도 참고할 수 있다. 내가 원하는 것은 외부 감지 센서 같은 것이 아니라, 한 번 지나가면서 전체를 검사하고 판단을 마칠 수 있는 디자인이다. 하지만 현실적으로는 제한적이다. 전체를 다 아우르기는 어렵고 일부 기능만 수행할 수 있다. 예를 들어 균열만 탐지하거나, 철근 부식을 감지하는 방식 같은 것이다.

이러한 기능을 활용해서 이동하면서 부식을 검사하거나 초음파를 발사해 내부에 빈 공간이 있는지를 확인할 수 있다고 한다. 몇 가지 기능을 섞으면 전체는 어렵더라도 일부는 확인할 수 있는 장치는 만들 수 있다고 생각한다. 그래서 이 아이디어를 가져왔는데 더 좋은 방법이 있는지 여쭤보고 싶었다.

나는 이전에 유니티로 콘솔을 만들었던 것을 이야기하면서 이런 방향으로 가도 좋겠다고 생각했다. 그래서 이번에는 공포게임을 반영해 아이디어를 가져왔다. 플레이어의 심리 반응, 호흡, 심박, 체온, 땀 등을 감지해 게임과 연동하는 것이다. 앉아서 하는 방식이 아니라 동적으로 하고 싶다면 발의 움직임을 측정해 데이터를 보내는 FSR 매트를 활용할 수 있다. 발에 가해지는 힘을 측정해 게임 속 소리로 반영하는 것이다. 발소리가 커지면 적이 이를 감지하거나, 손에 땀이 나거나 심박수가 증가하면 공포도가 높아졌다고 판단해 이벤트를 발생시키는 방식이다. 호흡 센서를 사용해 숨을 참으면 은신이 되는 기능도 가능하다.

첫 번째 아이디어는 이것이고, 확장하면 심박 센서를 추가해 공포 감지를 더 정밀하게 할 수 있다. 유니티와 OBS, 윈도우 환경을 연결해 공포 점수를 출력하거나 게임 로직을 강화하는 식으로 확장할 수 있다.

두 번째 아이디어는 기존 콘솔의 확장판이다. 총기만 움직이는 것이 아니라 몸의 움직임을 반영해 360도 시야를 구현하는 것이다. 자이로 센서를 몸에 착용해 회전량을 계산하고 유니티와 연결한다. 고글이나 360도 스크린을 활용해 몰입도를 높이고, 총기 반동이나 게임의 다양한 요소를 더할 수 있다.

세 번째 아이디어는 공포게임과 방송 환경을 연결하는 것이다. 스트리머가 게임을 할 때 시청자가 후원으로 미션을 거는 방식을 확장해, 트위치 API와 채팅 스트림을 수신해 시청자 의견을 반영하는 것이다. 예를 들어 심박수가 일정 이상 올라가면 공포 이벤트가 발생하는 방식이다. 다만 API 처리 방식은 구체적으로 확인해야 한다.

네 번째 아이디어는 장애인을 위한 통합형 게임 컨트롤러다. 시각·청각 장애가 있는 사용자를 위해 진동 피드백을 제공하거나, 손이 불편한 사용자는 다른 신체 부위를 이용해 조작할 수 있게 하는 것이다. ESP32 블루투스 모듈과 유니티를 연동해 누구나 사용할 수 있는 통합형 장치를 만드는 방향이다.

또 다른 의견으로는 모기나 해충을 자동으로 잡는 장치에 대한 아이디어도 있었다. AI와 비전 센서를 활용해 곤충의 움직임을 인식하는 방식인데, 이미 시중에 제품이 존재한다. 가격이 높고 구현이 어렵지만, 응용 가능성은 있다.

마지막으로는 구형 차량에 급발진 방지 장치를 추가하는 아이디어도 있었다. 브레이크와 엑셀을 구분하지 못해 사고가 나는 경우를 방지하는 것이다. 다만 이미 유사한 아이디어가 많이 나온 바 있어 차별성이 필요하다.

또 다른 아이디어로는 디지털 트윈을 기반으로 촉감 피드백을 주는 장치다. 사용자가 손을 움직이면 센서가 이를 인식해 시뮬레이션에 반영하고, 햅틱 피드백을 통해 촉감을 제공한다. 장갑에 진동 모터를 달아 손가락 끝에 반응을 주거나, 비전 모델을 활용해 포즈를 인식해 반응시키는 방식이다. 이를 통해 사용자는 화면 속 물체를 직접 만지는 것 같은 경험을 할 수 있다.

최근에는 로보플로우와 같은 플랫폼을 활용해 데이터를 어노테이션하고 학습시켜 비전 모델을 만들 수 있다. 데이터를 업로드해 라벨링하고, 학습·검증·테스트 데이터셋으로 나눠 학습시키면 응용할 수 있다. 이미지 증강 기법을 활용하면 더 다양한 학습 데이터셋을 만들 수 있다.

### <러브버그만 잡는 자동 레이저 시스템>

### <공포게임 → 센서를 활용한 공포 게임 플레이> - 윤서, 주형

- 발에 센서
- 온도 센서, 심박수 측정 → 공포 측정
- 유니티 활용해서 게임 로직 강화
- 플레이어의 발걸음, 심리 반응, 호흡 패턴을 실시간 감지하여 Unity 기반 게임에 "은신과 노출, 공포 트리거"를 동적으로 연동.

**→ 생각 :** 

**1. FSR 센서 매트 (발걸음/움직임 감지)**

- **센서**: FSR402 또는 Velostat 시트 기반 압력 감지 패드
- **설치 위치**: 발바닥 영역, 4~6점 분포
- **처리 방식**:
    - 압력 변화 빠르면 뛰기, 천천히면 걷기
    

**2. GSR 센서 + MLX90614 (공포감 감지)**

- **센서**:
    - GSR 센서 (손가락 or 손바닥에 부착)
    - MLX90614 (비접촉 온도센서, 손등 표면 온도 감지)
- **처리 방식**:
    - GSR ↑ or 온도 ↑ 시 → 공포 증가 → Unity에서 귀신 등장 / 화면 흔들림 / 효과음 삽입 등 트리거

**3. MPX 압력 센서 (복식호흡 감지)**

- **센서**: MPX5010 or MPXV7002 + 벨트 부착
- **설치 위치**: 복부 위에 감지벨트 착용 or 마스크/콧구멍 앞
- **처리 방식**:
    - 연속 압력 변화 → “숨쉬는 중”
    - 일정 시간 압력 유지 → “숨참 상태” → 은신 판정

### <360도가 가능한 현실 게임 인터페이스>

- 자이로센서(MPU6050)를 몸에 착용 (허리띠, 조끼 등)
- Z축 자이로 값 변화 → 좌우 회전량으로 계산
- ESP32에서 BLE HID 마우스 이동값 또는 시리얼로 Unity에 전송
- Unity 게임 내에서 이 입력을 마우스 움직임처럼 인식 → 시점 회전

### <**감각·입력 통합형 접근성 게임 컨트롤러**>

- 시각장애인 또는 지체 장애인이 진동을 통해 방향 정보를 감지하고,
- 자신이 편한 신체 부위(턱, 발 등)로 게임을 조작하는 다채로운 입력 방식을 제공하는
    
    → 양방향 감각·입력 통합형 게임 컨트롤러
    

**구성 요소**

| 구성 항목 | 설명 |
| --- | --- |
| 진동 피드백 장치 (출력) | 벨트 또는 조끼 형태로, 4~8개의 진동 모터 내장. 적 접근 방향, 사운드 방향, 게임 알림 등에 따라 진동 위치 달라짐 |
| 터치/압력 기반 스위치 (입력) | 사용자가 자유롭게 부착 가능 (예: 턱 밑, 발가락, 어깨 등). FSR, 정전식, 압력 기반 입력 가능 |
| ESP32 + BLE HID | 입력을 게임 키 입력으로 전송 / 진동 출력을 BLE 또는 시리얼로 수신 |
| Unity or Game Hook 연동 | 게임 내 이벤트를 수신하여 진동으로 전달, 입력은 HID 마우스/키보드로 전송 |

**작동 방식 예시**

**FPS 게임에서:**

- 적이 오른쪽에서 소리 내며 다가오면 → 오른쪽 진동 모터가 울림
- 유저가 턱 아래 터치 스위치를 눌러 총 발사, 발가락으로 이동 버튼 입력
- 상황에 따라 진동+압력 입력 조합으로 정밀 조작

**전략 게임에서:**

- 화면 특정 이벤트 → 방향 정보 없이 전면 진동
- 각 스위치는 유닛 이동, 공격 등 기능 수행

### <??? 상자> - 주형 형진

- 상자같은 사각형 환경 모서리에 센서들이 장착되어 있음.
- 상자에 손(변형 가능한 물체)이 들어오면, 뎁스 카메라를 이용하여 손의 형상을 땀
- 추출한 손을 시뮬레이션에 넣음 → 디지털 트윈
- 디지털 트윈을 통해 실제 손에 촉감을 느낄 수 있게 해줌
- 손으로 물체를 옮기면, 게임 상에 해당 물체가 움직일 수 있음
- URDF와 같은 구조
